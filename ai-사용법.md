# CLI (claude, chatgpt, gemini, grok 등) 플랜모드 사용법

**아래는 AI 수정 금지**

## 요약

```
명령어 요약

──────────────────────────────────────────────
[단계 1] 문서에 메모(주석) 추가 → AI에게 반영 명령
──────────────────────────────────────────────
plan.md 또는 관련 문서에 직접 메모를 작성한 뒤 아래처럼 명령한다.

> 문서에 몇가지 메모를 추가했어 모든 메모에 반영하고 문서를 업데이트해 아직은 구현하지마

→ AI는 메모를 읽고 계획/설계 문서만 업데이트. 코드 구현 X

──────────────────────────────────────────────
[단계 2] 문서 검토 완료 후 기능 개발 명령
──────────────────────────────────────────────
[plan.md, claude.md, research.md , **.**.md] 를 참고하여 기능 개발
표준 프롬포트:

> 전부 구현해라
> 작업이나 단계를 완료하면 계획 문서에서 완료로 표시해라
> 모든 작업과 단계가 완료될 때까지 멈추지 마라
> 지속적으로 typecheck를 실행해서 새로운 문제를 만들지 마라
> any나 unknown타입 쓰지마라
```

잘못된 경우 아래 순서 진행

```markdown
1. 불만족 하는 부분을 요구사항에 추가해서 계획 문서 업데이트
2. "git reset --hard HEAD" 작업 이전으로 커밋을 전부 되돌림
3. 이제 ~것만 그것 외엔 아무것도 하면안됨 *중요*
```

## md 구조

- `research.md` : 리서치 내용 작성
- `**/claude.md` : 모듈화된 리서치 내용 바탕으로 계획 작성
- `plan.md` : 현황 대시보드 (체크리스트 + 문서 링크)
- `infra.md` : 배포 / 인프라 셋업 절차
- `backend/CODE.md` : 코드 컨벤션 + 규칙 + 테스트 전략

---

## 목표

삽질제거 — 개발자의 주도권 확보
토큰 확보 — 결과물 향상

**작업 흐름:** 코드 리서치 → 계획 → 주석달기 → (반복) → 구현 → 피드백

발견한 내용은 반드시 `research.md`에 작성해야 함.
구현이 원하는대로 안 된다면 commit 리셋 후 위 흐름 반복.

---

## 명령 원칙

1. 채팅창에서 구두 요약으로 명령 X — 문서에 먼저 기록
2. `"깊이"`, `"매우 상세히"`, `"세부사항"` 단어가 중요함
   - Claude는 특성상 대충 보고 넘기는 경향이 있음
3. 범위를 좁히는 게 잘못된 접근을 조금씩 고쳐나가는 것보다 항상 더 좋은 결과를 만듦
4. 뭘 만들지 AI가 정하지 않음 — 기술 선택은 개발자가 함

## 방향 조정 패턴

1. 좋은 건 취하고 나머진 버리기
2. 과감하게 쳐내기
3. 건드리면 안 되는 선 긋기
4. 기술 선택은 내가 하기

---

## 가장 비싼 실패 유형

| 실패 | 원인 | 방지 |
|------|------|------|
| 기존 레이어 무시 | ORM 관례 무시, 마이그레이션 충돌 | 리서치 단계에서 기존 코드 먼저 파악 |
| 중복 API 엔드포인트 | 이미 있는 엔드포인트 재구현 | `backend/claude.md` API 목록 확인 |
| 설계도 없이 구현 | AI가 상상으로 채움 | plan.md 충분히 구체화 후 구현 |

**해결 방법**: 리서치 단계에서 설계도를 보완해야 함 (AI가 상상력이 0에 수렴한다고 생각할 때까지 수정)
